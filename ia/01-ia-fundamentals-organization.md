# 📐 01 — IA Fundamentals & Organization

> 🟢 Beginner | Prereq: — | ~7 min

Information Architecture (IA) = the structural design of shared information environments. It's how you organize, label, and connect content so users find what they need.

---

## 1.1 IA Definition (Rosenfeld & Morville)

| Component | Question |
|-----------|----------|
| **Organization** | How is content categorized? |
| **Labeling** | What do we call things? |
| **Navigation** | How do users browse? |
| **Search** | How do users find? |

> **The 3 Circles**: Context (business goals) + Content (what exists) + Users (what they need). IA sits at the intersection.

---

## 1.2 LATCH Principles (Richard Saul Wurman)

| Method | Example | Best For |
|--------|---------|----------|
| **Location** | Store finder, maps | Physical/spatial data |
| **Alphabet** | Contact list, glossary | Known-item seeking |
| **Time** | Calendar, timeline, changelog | Events, history |
| **Category** | Product catalog, topics | Browsing, exploration |
| **Hierarchy** | Org chart, nested menus | Importance, scale |

---

## 1.3 Organization Schemes

| Scheme | Description | Example |
|--------|-------------|---------|
| **Exact** | Objective, unambiguous | Alphabetical, chronological, geo |
| **Ambiguous** | Subjective, requires judgment | Topic, task, audience, metaphor |
| **Hybrid** | Mix exact + ambiguous | E-commerce: category (ambiguous) + price (exact) |

---

## 1.4 Organization Structures

| Structure | Shape | Use When |
|-----------|-------|----------|
| **Hierarchy** | Tree, parent-child | Most common — categories, folders |
| **Database** | Flat, tagged, filtered | Content with multiple attributes |
| **Hypertext** | Web, linked nodes | Wiki, knowledge bases |
| **Linear** | Sequential, step-by-step | Tutorials, wizards, checkout |

---

## 1.5 IA Heuristics

| # | Heuristic | Check |
|---|-----------|-------|
| 1 | **Findable** — Can users locate content easily? | Search test |
| 2 | **Accessible** — Can all users reach content? | WCAG audit |
| 3 | **Clear** — Do labels make sense? | Card sort |
| 4 | **Communicative** — Does IA match user mental models? | Tree test |
| 5 | **Usable** — Can users complete tasks? | Task testing |
| 6 | **Credible** — Is content trustworthy? | Content audit |
| 7 | **Learnable** — Do users improve with repeated use? | Retention test |
