# ⚖️ 01 — Ethical Frameworks for Designers

> **Level: 🟢 Beginner** | Prerequisite: — | ~8 min

Ethics isn't about avoiding "bad" design — it's about making deliberate value judgments. This course maps 4 major ethical frameworks to design decisions you make every day.

---

## 1.1 Four Frameworks — One Question Per

| Framework | Core Question | Design Translation |
|-----------|--------------|-------------------|
| **Utilitarian** | Does this produce the most good for the most people? | "Does infinite scroll serve users or engagement metrics?" |
| **Deontological** | Would I want everyone to design this way? | "If every app used dark patterns, would the web be usable?" |
| **Virtue Ethics** | What would an honest designer do? | "Would I be proud to explain this feature to my mom?" |
| **Care Ethics** | Does this strengthen human relationships? | "Does autoplay bring people together or isolate them?" |

## 1.3 Red Flag Checklist

When any of these are true, pause and reflect:

| Red Flag | Example |
|----------|---------|
| You're reluctant to explain the feature publicly | Hidden subscription renewal logic |
| Success metric is user manipulation, not satisfaction | "Time on site" at cost of user goals |
| The feature disproportionately harms vulnerable groups | Predatory lending UI targeting low-income users |
| You're copying a competitor's pattern without understanding why | "Everyone else does confirm-shaming" |
| Legal compliance is the ONLY reason you're doing it | GDPR consent but dark-patterned cookie banner |

---

## 1.4 Ethics in Practice — The ACM Code

| Principle | Design Application |
|-----------|-------------------|
| **1.1 Contribute to society** | Build tools that solve real problems, not addiction loops |
| **1.3 Be honest and trustworthy** | No hidden costs, no disguised ads, clear data practices |
| **1.4 Be fair and avoid discrimination** | Test with diverse users, audit algorithms for bias |
| **2.2 Maintain competence** | Stay current on accessibility, privacy, AI safety |
