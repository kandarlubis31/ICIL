# ♿ Faculty of Accessibility — Overview

> **7 Courses** | Level: Beginner → Advanced | Domain: Web Accessibility & Inclusive Design

This faculty covers the complete accessibility spectrum: from WCAG foundations (POUR principles, conformance levels, legal frameworks) through semantic HTML structure, ARIA roles/states/properties, keyboard navigation & focus management, accessible forms & inputs, screen reader testing methodologies, to advanced inclusive design patterns for cognitive, visual, motor, and auditory disabilities. Designed so AI agents can build, test, and audit digital products for genuine accessibility — not just compliance, but true inclusion.

## 🎯 Learning Paths

### Beginner Path (Foundations)
```
01 → 02
```
Understand WCAG principles and master semantic HTML — the bedrock of all accessibility.

### Intermediate Path (Implementation)
```
01 → 02 → 03 → 04 → 05
```
Add ARIA, keyboard navigation, and accessible forms — covers most WCAG AA requirements.

### Advanced Path (Full Accessibility)
```
01 → 02 → 03 → 04 → 05 → 06 → 07
```
Complete accessibility education: from foundations through testing to advanced inclusive design patterns.
