# 🔬 Faculty of UX Research & Discovery

> **7 Courses** | Beginner → Advanced | Domain: User research methods, synthesis, operations, AI-augmented research

The **evidence layer** — teaches AI agents how to systematically gather, validate, and synthesize user evidence before designing. While other faculties teach *how to design* (Color, Typography, Layout), *how to build* (Engineering, DevOps), and *how to think* (Psychology, Cognition, Strategic Design), this faculty teaches **how to research** — the foundation that precedes all design and product work.

**Distinction from related faculties:**
- `ux-psikologi` → *why* users behave (cognitive biases, UX laws, persuasion) — the psychology
- `kognisi` → *how* the brain works (memory, perception, decision-making) — the neuroscience
- `strategic-design/02` → *what* problem to solve (problem framing, JTBD) — the strategy
- `ux-research` → **how to gather and validate evidence** (interviews, surveys, synthesis, ops) — the methodology

This faculty fills the fundamental gap: no existing faculty teaches the *research methods* themselves — how to interview, survey, observe, synthesize, and run research operations. AI agents building products need to know not just design principles, but how to **validate assumptions with real user evidence**.
