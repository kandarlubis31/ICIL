# 🔧 01 — Improvement Methodology & Framework

> 🟢 Beginner | Prereq: — | ~3 min

Improvement = systematic discipline, not random tweaking. **Evidence over opinion.**

---

## 1.1 Core Principles

| Principle | Meaning |
|-----------|---------|
| Evidence over opinion | Every change justified by principle/data |
| Systematic over random | Follow structured process |
| Incremental over revolutionary | Small verified changes compound |
| Holistic over isolated | Color change affects contrast + hierarchy + a11y |
| Measure over assume | Track what changed and whether it improved outcomes |

---

## 1.2 5-Phase Improvement Cycle

```
AUDIT → ANALYZE → PRIORITIZE → EXECUTE → MEASURE → (repeat)
```

| Phase | Output |
|-------|--------|
| **Audit** | Categorized findings across 6 dimensions |
| **Analyze** | WHAT + WHY + WHO + principle + fix + faculty |
| **Prioritize** | Impact-Effort matrix (Quick Wins → Major → Fill-ins → Skip) |
| **Execute** | One change at a time, with principle reference |
| **Measure** | Verify: solve issue? regression? responsive? dark mode? a11y? |

---

## 1.3 Scope Assessment

| Scope | Approach |
|-------|----------|
| **Polish** | Surface fixes (contrast, spacing, microcopy), 1-2h |
| **Refinement** | Targeted redesign (forms, nav), 1-2d |
| **Overhaul** | Major rework (new DS, new IA), phased, 1-2w |
| **Rebuild** | Full restart with existing content, weeks |

---

## 1.4 Common Anti-Patterns

| Anti-Pattern | Why It Fails |
|-------------|-------------|
| Shotgun improvement | Change everything → can't isolate cause |
| Aesthetic-only | Pretty but still unusable |
| Copy-paste trends | Glassmorphism without considering contrast |
| Gold plating | Perfect shadows, broken navigation |
| No measurement | "I think it's better" ≠ proof |
