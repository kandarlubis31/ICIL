# ✍️ 06 — Accessibility & Inclusive Language

> 🔴 Advanced | Prereq: 01 | ~2 min

Accessible UX writing = ALL users understand, including those with disabilities, non-native speakers, diverse backgrounds. Not "nice to have" — mandatory.

---

## 6.1 Plain Language Rules

| Rule | Target |
|------|--------|
| Reading level | Grade 7-8 (middle school) |
| Sentence length | 15-20 words max |
| Paragraph | 2-3 sentences |
| Jargon, slang, idioms | **Avoid** |

```
❌ "Verify credentials to access activated premium functionality."
✅ "Log in first to access premium features."
```

---

## 6.2 Screen Reader Friendly

| Element | Rule | ❌ | ✅ |
|---------|------|-----|-----|
| Button/Link | Descriptive destination | "Click here" | "Download Q3 report" |
| Link text | Meaningful out of context | "For more info, click here." | "Read the full guide in Help Center." |

---

## 6.3 Inclusive Language Replacements

| Avoid | Replace With |
|-------|-------------|
| Guys (gendered) | Everyone, folks, team |
| He/she, him/her | They, them (singular) |
| Man, mankind | People, humanity |
| Normal user | Typical user |
| Disabled person | Person with disability |
| Blacklist/whitelist | Blocklist/allowlist |
| Master/slave | Primary/secondary, main/replica |
