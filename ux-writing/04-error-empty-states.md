# ✍️ 04 — Error & Empty States

> 🟡 Intermediate | Prereq: 01 | ~3 min

Error & empty states = most crucial UX moments. Good copy transforms frustration → relief, blank page → exciting start.

---

## 4.1 Error Message Formula: WHAT + WHY + SOLUTION

```
❌ "Error 404."              → Vague, no solution
✅ "Page not found. The link might be wrong or moved.
    [Back to Home] [Report Issue]"
```

| ❌ DON'T | ✅ DO |
|----------|------|
| "An error occurred." | "Wrong password. Try again or reset." |
| "Error: 0x8004DEF7" | "Connection lost. Check your internet." |
| "You filled the form wrong." (blaming) | "Email already registered. Log in instead?" |
| "Authentication failed." (jargon) | "File too large (max 10MB). Compress it." |

### Tone: Empathy first, solution second.
```
ROBOTIC:  "Invalid credentials. Authentication failure."
BLAMING:  "You entered the wrong password!"
GOAL:     "That password doesn't match. Check again or reset here."
```

---

## 4.2 Empty States — Golden Opportunity

| Type | When | Goal |
|------|------|------|
| First use | New user, no data | Guide + motivate |
| User cleared | Deleted all data | Reassure + encourage |
| No results | Search/filter empty | Help refine |
| Error | Data failed to load | Explain + retry |

```
❌ "No items found."         → Confused, stuck
✅ 🎨 "No projects yet!       → Guided, motivated
    Start your first project."
    [Create First Project]
```

## ⚡ Action Checklist
- [ ] Every error message answers: WHAT happened + WHY + HOW to fix
- [ ] Never blame the user — use neutral, helpful language
- [ ] Empty states have: illustration + encouraging copy + clear CTA
- [ ] Test all error messages: can a non-technical friend understand them?
