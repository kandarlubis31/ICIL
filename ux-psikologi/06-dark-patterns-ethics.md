# 🧠 06 — Dark Patterns & Design Ethics

> 🔴 Advanced | Prereq: 05 | ~3 min

Dark patterns = designs that **intentionally deceive** users. Now illegal in EU under DSA.

---

## 6.1 Dark Patterns Catalog

| Pattern | Description | Example |
|---------|------------|---------|
| **Roach Motel** | Easy in, hard to get out | 1-click signup, call-CS-to-cancel |
| **Bait & Switch** | Click X, something else happens | "Close" button opens new popup |
| **Hidden Costs** | Real price only at checkout end | +$15 "service fee" on last step |
| **Forced Continuity** | Trial charges without warning | No reminder before billing |
| **Confirmshaming** | Guilt for not converting | "No, I don't want to be successful" |
| **Trick Questions** | Deliberately confusing | "Don't NOT send me emails" double negative |
| **Privacy Zuckering** | Share more data than realized | Default=public, tiny "Customize" link |
| **Fake Urgency** | Resetting timers/scarcity | Countdown resets on refresh |

---

## 6.2 Dark Pattern vs Ethical Persuasion

| Aspect | Dark Pattern | Ethical Persuasion |
|--------|-------------|-------------------|
| Intent | Deceive / manipulate | Help / guide |
| User benefit | None (or harmful) | Win-win |
| Transparency | Hidden | Clear & open |
| User autonomy | Reduced | Preserved |
| Long-term | Trust destroyed | Loyalty grows |

---

## 6.3 The "Mom Test"

> If you'd be embarrassed to explain this tactic to your mom → it's a dark pattern.
