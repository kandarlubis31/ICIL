# 🧠 01 — UX Psychology Foundations

> 🟡 Intermediate | Prereq: — | ~4 min

Brain processes 11M bits/sec but only ~50 consciously. UX designers are **attention architects** — you determine which 50 bits enter consciousness.

---

## 1.1 System 1 vs System 2 (Kahneman)

| | System 1 | System 2 |
|---|---------|---------|
| Speed | Fast, instant | Slow, analytical |
| Energy | Low (automatic) | High (effortful) |
| Awareness | Subconscious | Conscious |
| UX context | Casual browsing, scrolling | Complex forms, setup |
| Design rule | **Design for this first** | Support when needed |

> Golden Rule: Design for System 1 (make it easy), support System 2 (help with complex decisions).

---

## 1.2 Mental Models

User's internal picture of how something works, from past experience. **If UI matches mental model → intuitive. If it violates → confused.**

| Icon | Mental Model |
|------|-------------|
| 🗑️ Floppy disk | Save |
| 🔍 Magnifying glass | Search |
| 📧 Envelope | Email |
| ⚙️ Gear | Settings |

> **Jakob's Law**: Users spend most time on OTHER sites — they prefer yours to work the same way.

---

## 1.3 Memory Types

| Type | Duration | Capacity | UX Example |
|------|----------|----------|-----------|
| **Sensory** | <1 sec | Large | Motion detection, flicker |
| **Short-term (Working)** | 10-30 sec | 4-7 items | OTP code, form fields |
| **Long-term** | Unlimited | Unlimited | Password recall, brand recognition |

**Recognition > Recall**: Dropdown/menu (recognize) beats free-text command (recall). Never force users to remember.

---

## 1.4 Cognitive Load

| Type | Definition | UX Goal |
|------|-----------|---------|
| **Intrinsic** | Inherent task complexity | Simplify |
| **Extraneous** | Bad design distractions | **ELIMINATE** ← main enemy |
| **Germane** | Learning/understanding energy | Optimize |

Extraneous red flags: excessive animations, autoplay video, stacked popups, inconsistent nav, jargon.

---

## 1.5 Attention & Decision Making

| Fact | Implication |
|------|------------|
| Attention span: ~8 sec | Visual hierarchy: headline(0-3s) → subhead(3-5s) → body |
| Stay-or-leave: 0.05 sec | First impression = visual, not text |
| Users scan (F/Z pattern) | CTA above fold, thumb zone on mobile |
| Most users = satisficers | Limit choices, give defaults, "Most Popular" labels |

| Mode | Characteristic | UX |
|------|---------------|-----|
| **Maximizer** | Seeks the BEST | 50-item dropdown = nightmare |
| **Satisficer** | Seeks GOOD ENOUGH | 3-5 options = optimal |

---

## ⚡ Action Checklist
- [ ] Design main flows for System 1 (autopilot) — minimize cognitive effort
- [ ] Match UI patterns to existing mental models (don't reinvent the wheel)
- [ ] Prefer recognition (dropdowns, galleries) over recall (free-text commands)
- [ ] Eliminate extraneous cognitive load: autoplay, popups, jargon, inconsistency
- [ ] Limit choices to 3-5 items + "Most Popular" default for key decisions
