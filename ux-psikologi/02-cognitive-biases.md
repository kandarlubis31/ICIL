# 🧠 02 — Cognitive Biases in UX

> 🟡 Intermediate | Prereq: 01 | ~4 min

Cognitive biases = mental shortcuts that make decisions not always rational. Design to align with how the brain works.

---

## 2.1 Decision Biases

| Bias | Definition | Ethical UX Tactic | ⚠️ Dark Pattern |
|------|-----------|-------------------|----------------|
| **Anchoring** | First info seen anchors decisions | Show most expensive package first → rest look cheaper | Fake "original" prices |
| **Framing** | Presentation changes perception | Loss frame: "Don't miss out" > "Get this" | — |
| **Loss Aversion** | Pain of losing = 2× joy of gaining | "Your trial ends in 2 days" | — |
| **Choice Paralysis** | More choices = harder to decide | 3-5 options + "Most Popular" label | — |
| **Default Effect** | Users rarely change defaults | Default = best-for-user option | Pre-checked newsletter opt-in |

---

## 2.2 Perception Biases

| Bias | Definition | UX Application |
|------|-----------|---------------|
| **Aesthetic-Usability** | Pretty = perceived as more usable | Invest in visuals, but never skip function |
| **Von Restorff** | Different item = more memorable | CTA contrasts from all other elements |
| **Serial Position** | Remember first & last items best | Key items at navbar edges; pricing extremes |
| **Mere Exposure** | Familiarity → liking | Visual consistency builds trust over time |

---

## 2.3 Action & Social Biases

| Bias | Tactic | Example |
|------|--------|---------|
| **Social Proof** | Show others' behavior | "5,000+ companies use this", testimonials |
| **Scarcity** | Limited = valuable | "Only 3 left" (must be GENUINE) |
| **Authority** | Expert endorsement | Trust badges, certifications, "Recommended by" |
| **Commitment** | Small ask → escalate | Email → profile → upgrade |
| **Zeigarnik** | Unfinished = remembered | "Profile 70% complete" progress bar |
| **Peak-End Rule** | Judge by peak + ending | End flows with celebration/reward |

---

## ⚡ Action Checklist
- [ ] Set default options to what's best for the USER (not company)
- [ ] Use loss aversion phrasing: "Don't lose X" > "Gain Y"
- [ ] Verify all scarcity/urgency claims are genuine (no resetting timers)
- [ ] Show social proof (numbers, testimonials, real-time activity)
- [ ] End every critical flow on a positive note (Peak-End Rule)
